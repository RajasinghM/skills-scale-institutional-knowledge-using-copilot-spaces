# OctoAcme — Roles & Handoffs Checklist

## Purpose
This checklist ensures clear role ownership, communication cadence, and handoff points are established at project kickoff. Complete this during or immediately after the project initiation phase.

## When to Use
- At the start of any new project or major initiative
- When onboarding new team members mid-project
- When reorganizing team structure or responsibilities

---

## Staffing & Role Ownership

- [ ] **Core roles assigned** — Identify and confirm availability of:
  - [ ] Developer(s)
  - [ ] Product Manager
  - [ ] Project Manager
- [ ] **Additional roles assessed** — Determine if the following roles are needed:
  - [ ] QA Lead
  - [ ] Business Analyst
  - [ ] Technical Writer
  - [ ] Scrum Master (or PM covers Agile facilitation)
  - [ ] UX Designer
- [ ] **Primary contact documented** for each role (name, email, Slack/Teams handle)
- [ ] **Backup/coverage** identified for critical roles
- [ ] **Role interactions clarified** — Review [octoacme-roles-and-personas.md](octoacme-roles-and-personas.md) to understand dependencies and handoffs

---

## Communication Cadence

- [ ] **Daily standup** scheduled (time, frequency, participants)
- [ ] **Weekly status sync** scheduled (PM, PdM, key stakeholders)
- [ ] **Sprint planning / iteration planning** cadence agreed upon
- [ ] **Demo/Review** schedule established (end of sprint or milestone)
- [ ] **Retrospective** schedule established (every sprint or at major milestones)
- [ ] **Communication channels** set up:
  - [ ] Slack/Teams channel or equivalent
  - [ ] Email distribution list (if applicable)
  - [ ] Shared document repository (e.g., GitHub repo, wiki)

---

## Quality Responsibilities

- [ ] **QA approach defined** — Who owns test planning and execution?
  - [ ] QA Lead assigned OR
  - [ ] Developers own testing with PM oversight
- [ ] **Definition of Done (DoD)** includes quality criteria (unit tests, integration tests, code review, etc.)
- [ ] **Acceptance criteria** validation process agreed upon (QA Lead, PdM, or BA)
- [ ] **Bug triage process** established (who triages, frequency, severity levels)
- [ ] **Test environment** ownership and access confirmed

---

## Documentation Responsibilities

- [ ] **Technical documentation** ownership assigned:
  - [ ] Technical Writer assigned OR
  - [ ] Developers responsible for inline and API docs
- [ ] **Release notes** ownership assigned:
  - [ ] Technical Writer, Product Manager, or Project Manager
- [ ] **User-facing documentation** ownership assigned (if applicable)
- [ ] **Documentation review process** established (who reviews, cadence)
- [ ] **Knowledge base / wiki** location and maintenance plan confirmed

---

## Escalation Paths

- [ ] **Level 1 (Team-level)** — Daily standup or direct message to PM
- [ ] **Level 2 (Cross-team / dependency)** — PM escalates to Product Lead or dependent team leads
- [ ] **Level 3 (Business-impacting / critical)** — PM escalates to Sponsor or executive stakeholder
- [ ] **Escalation contact list** documented with names and response expectations
- [ ] **Escalation criteria** defined (what constitutes Level 2 vs. Level 3)

---

## Handoff & Dependency Points

- [ ] **Requirements handoff** — Business Analyst or Product Manager → Developers
  - [ ] Acceptance criteria clear and documented
  - [ ] User stories or specs reviewed and approved
- [ ] **Design handoff** — UX Designer → Developers
  - [ ] Design assets and specifications shared
  - [ ] Design review completed
- [ ] **Development to QA handoff** — Developers → QA Lead
  - [ ] Feature complete and deployed to test environment
  - [ ] Test cases identified or test plan updated
- [ ] **QA to Release handoff** — QA Lead → PM / Release Manager
  - [ ] Acceptance criteria validated
  - [ ] Release notes drafted
  - [ ] Known issues documented
- [ ] **Release to Support handoff** — PM / Technical Writer → Support Team
  - [ ] Release announcement sent
  - [ ] Documentation updated
  - [ ] Support team briefed on changes

---

## Sign-off

- [ ] **Checklist reviewed** by Project Manager and Product Manager
- [ ] **Team notified** of finalized roles, communication cadence, and escalation paths
- [ ] **Checklist stored** in project repository for reference
